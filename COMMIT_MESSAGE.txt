Complete backend implementation to replace localStorage with proper database.
All endpoints tested and verified working via PowerShell.

## Backend Infrastructure:
- PHP REST API with PDO database connection
- MySQL database (quiz_app) with 5 normalized tables
- CORS configured for React dev server (localhost:5173)
- Password hashing with bcrypt for security
- Prepared statements for SQL injection protection

## Database Schema:
- users: Authentication and profiles (instructors + students)
- quizzes: Quiz definitions with JSON questions column
- submissions: Student quiz attempts with tamper-proof storage
- results_released: Per-quiz result release control
- notifications: Auto-created on submission/release events

## API Endpoints (14 total):

### Authentication:
- POST /api/auth/login.php - User login with password verification
- POST /api/auth/register.php - User registration with bcrypt hashing

### Instructor Quiz Management (NEW):
- POST /api/instructor/create-quiz.php - Create quiz with all question types
- GET /api/instructor/get-quizzes.php - Get instructor's quizzes with stats
- POST /api/instructor/edit-quiz.php - Update quiz (owner verification)
- POST /api/instructor/delete-quiz.php - Delete quiz (cascade submissions)
- GET /api/instructor/submissions.php - View all student submissions
- POST /api/instructor/release-results.php - Release results per quiz

### Student Quiz Flow:
- GET /api/quiz/get-quizzes.php - List all available quizzes
- GET /api/quiz/get-questions.php - Get quiz with questions
- POST /api/quiz/submit.php - Submit answers with server-side scoring
- GET /api/quiz/check-submission.php - Check submission status

### Student Results:
- GET /api/student/results.php - Get student's quiz results

### Notifications:
- GET /api/notifications/get.php - Get user notifications
- POST /api/notifications/mark-read.php - Mark notification as read
